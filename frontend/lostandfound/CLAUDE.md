CLAUDE.md

Project Overview

This is a React + Vite (JavaScript) frontend for a Lost & Found web application built during a hackathon.

Your job is to build and improve only the frontend unless explicitly instructed otherwise.

---

Rules

- You can refer access .agents skills folder for ui design
- Never modify the "backend" folder.
- Work only inside the "frontend" folder.
- Never break existing functionality.
- Keep commits and changes focused on the requested task.
- Do not add unnecessary dependencies.
- Keep the project beginner-friendly and easy for teammates to understand.

---

Tech Stack

- React
- Vite
- JavaScript (No TypeScript)
- Tailwind CSS
- Framer Motion
- React Router

---

Project Structure

- Keep pages in separate folders (e.g. "pages/Home", "pages/Lost", "pages/Found").
- Use reusable components.
- Keep layouts separate from pages.
- Store dummy data separately.
- Organize assets, hooks, utils, constants, and components into dedicated folders.

---

UI Guidelines

Always refer to "DESIGN.md" before creating or modifying UI.

Follow its:

- Typography
- Colors
- Spacing
- Animations
- Layout
- Component styling
- Overall design language

Do not copy designs exactly—use them as inspiration.

---

Coding Guidelines

- Prefer reusable components over duplicated code.
- Keep components small and focused.
- Use descriptive file and component names.
- Keep imports clean.
- Remove unused code.
- Write readable, maintainable JavaScript.

---

Routing

Current routes:

- "/"
- "/lost"
- "/found"

Do not add additional routes unless requested.

---

Scope

Only implement what is explicitly requested.

Do not build:

- Backend logic
- Authentication
- API integration
- Database functionality
- Features not requested

If something is outside the current task, leave a clean placeholder instead of implementing it.

---

Goal

Maintain a scalable, production-ready frontend that multiple developers can work on simultaneously while keeping the UI consistent with "DESIGN.md".
